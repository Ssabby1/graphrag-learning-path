# 系统架构说明

## 1. 架构概览
系统采用 B/S 架构，由前端、后端、Neo4j 图数据库三部分组成。

## 2. 模块职责
- 前端：图谱可视化、目标选择、状态更新、路径展示。
- 后端：API 编排、路径算法、数据访问封装。
- Neo4j：知识点实体与先修关系持久化。

## 3. 数据流
1. 前端请求图谱与知识点详情。
2. 后端查询 Neo4j 返回图数据。
3. 用户选择目标后调用推荐接口。
4. 后端执行前驱检索 + 拓扑排序并返回路径。
5. 前端渲染路径并支持学习状态更新。

## 4. 可扩展点
- 支持多课程图谱。
- 增加推荐解释模块。
- 增加实验指标与看板。
