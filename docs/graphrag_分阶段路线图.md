# GraphRAG 项目分阶段实施路线图（大局版）

## 1. 项目总目标
构建一个面向课程学习路径推荐的 GraphRAG 系统，实现“可检索、可推理、可解释、可评估、可部署”。

核心能力目标：
1. 图谱化知识组织（结构化先修关系）
2. 混合检索（图 + 向量 + 关键词）
3. 证据约束生成（可解释输出）
4. 工程化服务（稳定 API、监控、测试、部署）

---

## 2. 总体技术架构（建议统一）
- 后端框架：FastAPI
- 图数据库：Neo4j
- 向量能力：Neo4j Vector Index 或外部向量库（FAISS/Chroma）
- 关键词检索：BM25/倒排索引
- 前端：Vue 3 + ECharts
- 任务脚本：Python（数据、索引、评估）

---

## 3. 分阶段目标与工作内容

## 阶段 A：数据与知识建模层
目标：建立高质量、可维护的知识图谱数据底座。

主要工作：
1. 定义统一数据模型
- 实体：Course/Chapter/Concept
- 关系：PREREQUISITE_OF 及扩展关系
- 证据字段：evidence_text/source/confidence

2. 建立数据处理流水线
- 原始数据标准化
- 去重、别名归一、关系方向校验
- 图导入与增量更新策略

3. 建立数据质量门禁
- 环检测、自环检测、重复边检测
- 必填字段完整性检查
- 质量报告自动化产出

阶段产出：
- 稳定的图谱 schema 与导入规范
- 可复用的数据校验机制
- 可追踪的数据版本

---

## 阶段 B：检索与索引层
目标：形成可扩展的 GraphRAG 检索能力。

主要工作：
1. Chunk 体系设计
- 概念块（概念描述、章节、别名）
- 关系证据块（边语义与证据文本）
- 元数据规范（concept_id、edge_id、confidence）

2. 索引构建
- 图索引：Neo4j schema/index/constraints
- 向量索引：embedding + ANN 索引
- 关键词索引：BM25 倒排

3. 混合检索策略
- 图召回：先修子图扩展
- 向量召回：语义近邻
- 关键词召回：术语精确命中
- 融合排序：权重融合 + 规则重排

阶段产出：
- 可离线重建的多索引体系
- 可在线调用的混合检索服务
- 可调参的召回/排序策略

---

## 阶段 C：推理与生成层
目标：实现“证据驱动”的可解释推荐输出。

主要工作：
1. 推荐推理流程
- 输入规范化（目标知识点、已掌握集合、学习目标）
- 路径候选生成
- 证据集合冻结（只允许证据内生成）

2. 提示词工程
- 系统提示：角色、边界、禁止幻觉
- 结构化输入模板：路径候选 + 证据表
- 结构化输出模板：path/evidence/reasoning/explanation/confidence

3. 生成结果约束
- 事实校验：概念与边必须可回溯到图谱
- 不确定性表达：证据不足时降级说明
- 自动回退：模板化解释兜底

阶段产出：
- 稳定、可审计的生成结果
- 可解释字段标准化输出
- 幻觉控制与回退机制

---

## 阶段 D：服务化与 API 层（FastAPI）
目标：对外提供稳定、清晰、可测试的服务接口。

主要工作：
1. FastAPI 服务设计
- 路由分层：health / graph / concept / recommend / admin
- DTO/Schema：请求响应模型强约束
- 依赖注入：配置、数据库、检索器、生成器

2. 核心 API 设计
- `GET /health`
- `GET /graph/overview`
- `GET /concept/{concept_id}`
- `POST /path/recommend`
- `POST /admin/index/rebuild`（运维接口）

3. 工程能力
- 统一错误码与异常处理
- 请求日志与耗时统计
- 并发/超时/重试策略

阶段产出：
- 可交付的 FastAPI 后端服务
- 清晰 API 文档（OpenAPI）
- 稳定的线上调用体验

---

## 阶段 E：前端产品化层
目标：把检索与推理结果可视化为可用产品体验。

主要工作：
1. 交互设计
- 目标知识点输入与学习状态输入
- 推荐路径可视化（图谱高亮）
- 证据与解释并排展示

2. 结果可解释展示
- 路径步骤卡片
- 每一步证据来源
- 置信度与替代路径

3. 可用性增强
- 错误兜底提示
- 历史记录与对比
- 移动端与桌面端自适应

阶段产出：
- 面向演示与答辩的完整前端流程
- 可理解、可复核的推荐结果展示

---

## 阶段 F：评估、优化与运维层
目标：让系统具备可量化优化能力和稳定运行能力。

主要工作：
1. 检索评估
- Recall@K、MRR、nDCG
- 图召回与向量召回贡献度分析

2. 生成评估
- 事实一致性（是否可回溯证据）
- 解释质量（清晰度、完整性）
- 幻觉率与回退率

3. 系统运维
- 数据更新流程（定时/手动）
- 索引重建流程
- 日志、告警、性能监控

阶段产出：
- 可持续迭代的指标体系
- 线上稳定性保障机制
- 版本化发布能力

---

## 4. 里程碑视角（建议）
1. M1：完成知识图谱数据底座与质量门禁
2. M2：完成混合检索与可调排序
3. M3：完成证据约束生成与可解释 API
4. M4：完成 FastAPI 服务化与前端联调
5. M5：完成评估闭环与部署交付

---

## 5. 技术选型原则（用于阶段决策）
1. 先可用再复杂：优先保证可运行闭环
2. 先可解释再高分：证据可追溯优先于“看起来聪明”
3. 先稳定再扩展：接口契约和数据质量先固定
4. 组件可替换：向量库、模型、检索策略都应可插拔

---

## 6. 一句话总结
这个项目应按“数据底座 -> 检索中台 -> 生成引擎 -> FastAPI 服务化 -> 前端产品化 -> 评估运维”六层推进，每一阶段都要有明确产出物与可验收指标。
